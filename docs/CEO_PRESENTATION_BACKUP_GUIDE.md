# Mighty Skill-Bridge CEOプレゼン障害時バックアップ＆フェイルオーバーガイド

本ガイドは、2026年6月2日の社長プレゼンおよびデモ実行時に、ネットワーク障害、APIエラー、外部ツールのトラブルが発生した場合に迅速に対処するためのバックアップ手順をまとめたものです。

---

## 1. 公開デモURL障害時のフェイルオーバー（ローカル起動手順）

もし GitHub Pages にデプロイされた公開URL [https://kanta13jp1.github.io/mighty-link-ai-connect/](https://kanta13jp1.github.io/mighty-link-ai-connect/) がネットワーク切断等の理由で開けない場合、ローカル環境でモックアップサーバーを即座に起動してデモを継続します。

### ■ ローカル起動コマンド

1. リポジトリのルートディレクトリで、PowerShellを開きます。
2. 以下のコマンドを実行して、AIモックを強制した状態でローカルサーバーを起動します。

   ```powershell
   $env:AI_FORCE_MOCK="1"
   python src/app.py
   ```

   *(※ Linux/macOS等の bash 環境の場合は `AI_FORCE_MOCK=1 python src/app.py` を実行)*
3. 起動後、ブラウザで [http://127.0.0.1:8000/](http://127.0.0.1:8000/) にアクセスします。
4. これにより、完全にローカルかつ無停止で Seedance風 UI デモが正常動作します。

## 2. 静的バックアップ資料の活用（完全オフライン時）

PCやサーバーの機能が完全に停止している、あるいはローカル起動が不可能な場合の最終手段として、あらかじめ撮影された高解像度スクリーンショットおよび各種仕様書PDFを活用します。

- **静的スクリーンショットの保存場所**:
  `exports/knowledge_flow/screenshots/` 配下に、フロントエンドUIの各状態（分析入力、分析中、結果表示、レーダーチャート等）の画像が格納されています。
- **PPTX プレゼン資料の活用**:
  ローカルおよびGoogle Drive上の `mighty_skill_bridge_ceo_presentation_2026-06-02_branded.pptx` には、フロントエンドのデザインワイヤーフレームやアーキテクチャ図が埋め込まれています。スライドを映すだけでデザインクオリティを十分にアピールできます。

## 3. Googleカレンダー障害時の ICS手動インポート手順

Google Calendar APIの不具合やネットワーク規制により、開発計画カレンダー（Googleカレンダー）が同期されない、または表示されない場合、ローカルに書き出されているICSファイルから手動でカレンダーを復旧・再現します。

### ■ ICS手動インポート手順

1. パソコンで [Google カレンダー](https://calendar.google.com/) を開きます。
2. 右上の設定アイコン（歯車マーク） ➔ **[設定]** をクリックします。
3. 左側のメニューで **[インポート / エクスポート]** をクリックします。
4. **[パソコンからファイルを選択]** をクリックし、リポジトリ内の以下のファイルを選択します。
   - `exports/mighty_development_plan.ics`
5. インポート先として、復旧用またはデモ用に新しく作成したマイカレンダー（例: 「Mighty Skill-Bridge 開発計画（代替）」）を選択します。
6. **[インポート]** をクリックします。
7. 数秒でカレンダー上に全タスクの予定（開始〜終了）が綺麗に再現されます。

## 4. Google Sheets 障害時のローカル代替対応

スプレッドシート（WBSや課題管理等）がGoogleのサービスダウンにより開けない場合、ローカルのTSVファイル群をExcelやCSVエディタで開き、プロジェクターに投影して進行します。

- WBSデータ: `data/WBS.tsv`
- 課題管理表: `data/issues_tracker.tsv`
- QAリスト: `data/qa_tracker.tsv`
- テスト結果: `data/test_results.tsv`
- セキュリティログ: `data/security_log.tsv`
- デプロイ実行ログ: `data/deploy_log.tsv`
